Design notes
  • Tone: friendly, not punitive. Avoid words like "Forbidden," "Access denied," "401/403."
  • Always offer a path forward: primary button takes them somewhere they CAN go (their school, their home, their profile).
  • Always offer escape: "email talent@" link for "this is wrong, fix me" cases.
  • Variant A vs B is the key distinction — Variant A is "you have access just not THIS school"; Variant B is "you don't have this kind of access at all."
  • Pages to wrap: /app/network, /app/network/* (drill-downs), /app/network/school/* (school deep-dive), /app/admin/* (impersonation, permissions matrix).
  • API behavior unchanged: backend still returns proper HTTP 403; frontend Router catches the auth response and renders one of these screens. No change to logging or audit.
