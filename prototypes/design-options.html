Observation Form — 6 options
Accessing recent data while scoring
You're completing an observation on Marcus. Five ways to bring recent data into the form — all user-initiated (choice to see), none forced. Goal: help the observer be fair and connected without leaving the task.
N
Context Strip + Drawer
Thin strip at top · slide-up drawer
Always-visible one-line context ("Last obs Jan 14 · Cycle 3 closed"). "Review recent" button opens a slide-up drawer with last 4 touchpoints + open cycle. Mobile-native feel.
Open →
O
Inline Per-Dimension
"Recent" button on each row
Each dimension has its own "Recent" toggle that expands a mini-history just for that dimension — scores, notes, tiny sparkline. Granular context exactly where you need it.
Open →
P
Side Panel / Bottom Sheet
Dedicated surface alongside
Floating "Review recent" button opens a side panel (desktop) or bottom sheet (mobile). Form stays live, panel has filters for touchpoint types. For deeper review.
Open →
Q
Pre-Form Splash
30-second prep screen
Before the form loads, an optional summary screen: hero quote, YoY grid, last 3 touchpoints, open cycle status. "Start observation" button. Dismissible "always skip."
Open →
R
Ambient Last Score
Sparkline + last score on each row
Persistent mini-sparkline + last-score chip next to every dimension. Zero clicks to see trajectory, but can be toggled off with one switch if you want a clean slate.
Open →
S
Whisper
One line, rotating hints
Minimalist pinned strip cycles through 4 one-line signals ("CoL held at 3 for 5 touchpoints"). Dot navigation. "Review ↓" expands to full history. Quietest option.
