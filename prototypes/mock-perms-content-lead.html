Why these are hidden
  • PMAP — annual evaluation about someone else, tied to compensation/retention. Belongs to direct supervisor + HR.
  • PIP — formal corrective action. Belongs to direct supervisor + HR.
  • Write-Up — disciplinary record. Same as PIP.
Why Self-Reflection IS visible
  • SR is universally self-only — every staff member completes their own.
  • The form auto-routes to the right variant (Teacher / Leader / Network / Support / PreK) based on Sarah's job role.
Open questions
  • Staff profile view — when Sarah taps into a teacher, the profile has many sections. Hide just the PMAP/SR/PIP/Write-Up sections, or hide the whole "Personnel" tab?
  • Sarah currently has admin access via the "ExDir of Teach and Learn" / K-8 Content Lead title not matching ExDir/Chief — actually K-8 Content Lead doesn't match C_TEAM_KEYWORDS so she'd be a regular supervisor today. ExDir of Teach and Learn DOES match ExDir keyword. So the practical scope of this tier today: only ExDir of Teach and Learn needs the override; K-8 Content Lead would already be supervisor-level.
  • Should Content Leads see PIP form on Home for teachers in their content area, even if read-only? (probably no — keep clean separation)
Backend changes
  • Order of role checks matters — is_content_lead() must run BEFORE is_admin() so ExDir of Teach and Learn gets narrowed scope, not full admin
  • /api/auth/status returns a 'role_bucket' field so frontend can hide PMAP/PIP tiles + Evaluations strip without the user noticing they're missing
  • Server-side enforcement: /api/network/drilldown?kind=evaluations returns 403 (with friendly screen) for content_lead
