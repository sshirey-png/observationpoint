StaffProfile · Action Steps · Before vs After

What we're solving: Today, the Action Steps section on a staff profile is hardcoded to school_year === '2025-2026'. If a viewer comes from a Network drill-down set to a different year, the drill-down counts steps but the profile filters them all out → "No action steps."

Goal: The profile follows the year the user was viewing on Network. Click into Jeffrey from a 25-26 drill-down → his 25-26 work is focused. Click in from 26-27 → 26-27 is focused. Other years still accessible (collapsed) so historical context isn't lost. Each step card carries a year badge.

What changes
  • Focused year follows the URL. Click into Jeffrey from a 25-26 drill-down → 25-26 is focused. Click in from 26-27 → 26-27 is focused. Drill-down nav now appends ?sy=25-26 to the staff URL.
  • Focused year shows expanded with active step cards. Mastered + Not Mastered collapse into disclosure rows so the page isn't noisy with completed work.
  • Other years collapse to single rows with the count (e.g. 2026-2027 (3) ›). Expand to see those steps with their original state badges.
  • Year badge on every step so when you do expand another year, each step reads contextually — you don't mistake a 2-year-old "In Progress" for current work.
  • Nav subtitle reads "Staff Profile · 2025-2026" so it's obvious which year the page is anchored to.
  • Fallback: if the URL has no sy, default to the newest year that has any data for this teacher.
  • Empty states: "No action steps yet" only if zero across all years. If there's history but nothing in the focused year, message reads "No action steps for 2025-2026."
